Background

How GrammarX was born

The starting point
English 2 class at UPDS required a group project. The team of four Systems Engineering students decided to build something they would actually want to use.
The observation
Grammar exercises in class were repetitive and passive. Students — including ourselves — completed them without real engagement or motivation to improve.
The decision
Inspired by classroom quiz tools like Kahoot, we set out to build our own — focused exclusively on English grammar, with more depth and built entirely from scratch.
The result
A fully functional multiplayer web game, deployed to the cloud and accessible globally — with no templates, no pre-made platforms, built line by line by our team.

The Challenge

Why does grammar feel boring?

😴
Passive, repetitive exercises — students fill in blanks and complete worksheets mechanically. There is no urgency, no competition, and no real incentive to think carefully.
⏳
No immediate feedback — corrections arrive long after the mistake was made. The connection between error and correction is lost, so concepts fail to stick.
🏝️
Students work in isolation — there is no social or competitive element. Without a reason to outperform others, motivation stays consistently low.
📉
Low retention of grammar rules — without repeated, varied, engaging exposure, students memorize for the test and forget shortly after.
"Students don't need more grammar worksheets — they need a reason to care and a way to see their progress in real time."
Our Answer

Introducing GRAMMARX

Gamification meets English grammar practice.
🏆
Live classroom competition — the whole class plays simultaneously, each student racing to answer first and score the most points.
⚡
Instant per-answer feedback — correct or wrong is shown immediately after each answer, with a speed bonus that rewards both accuracy and fast thinking.
🎮
Teacher controls the room — the host selects the game mode, sets the time per round, reads results live, and advances each round manually.
🔀
Always unique sessions — ten questions drawn randomly from a 40+ item bank per difficulty level, so repetition is never exact.

Game Modes

The 4 Mini-Games

Each mode targets a different grammar structure · 5 difficulty levels · 40+ questions per mode
⏰
Time Machine
Past Simple · Word Order
The words of a Past Simple sentence are scrambled. Tap them in the correct order before time runs out. Covers regular and irregular verbs.
⚔️
Duel Mode
Comparatives & Superlatives
Two things are compared. Choose the correct comparative or superlative form of the adjective from 4 options. Short adj, long adj, irregular forms.
🔍
Evidence File
Present Perfect · Error Detection
A Present Perfect sentence with one grammar error is shown in red. Identify the mistake and select the corrected version from 4 options.
🔮
Crystal Ball
Future Tense
Read the situation. Choose the correct future form — will, going to, or won't — to complete the sentence. Plans, predictions, and decisions.
Game Flow

How to Play

No installation required · No sign-up · Open in any browser
①
Teacher creates a room
A unique 6-character alphanumeric code and a QR code are generated automatically. Project the code on the board or share the QR.
②
Students join from any device
Go to grammarx.onrender.com · Enter your name · Type the room code or scan the QR with your phone camera. No account needed.
③
Host selects game mode & time per round
Choose one of 4 mini-games · Set the timer (10 – 60 seconds per question) · Everyone reads the instructions · Press ▶ START when all are ready.
④
10 rounds — fastest & most accurate player wins
Points = base score × difficulty + speed bonus. The host advances each round. Final podium shows top 3 players. Play again instantly with a new game mode.

Real-Time Communication

How real-time sync works

The state flow
Player answers → socket.emit('answer') ↓ server validates + stores in SQLite ↓ push(code) is called ↓ io.to(code).emit('state', newState) ↓ All clients update their UI instantly
Reconnection
If a player disconnects mid-game, they emit 'rejoin' on reconnect. The server sends back the current state immediately — no progress is lost.
Socket.io rooms
Each session is a Socket.io room identified by the 6-char code.

When any event occurs (answer, round advance, game end), the server calls push(code) which broadcasts the updated state to every socket in that room with one line:

io.to(code).emit('state', state)
Timer synchronization
The countdown runs client-side for smooth UI, but the server timestamps each round start. On round advance the server computes final scores including the speed bonus from timeLeft / timePerRound.
Technology Choices

Tech Stack — Why each tool

⚛️ React 18 + Vite 5
Frontend SPA
React re-renders only affected components when state changes — critical for a game where the timer, answer counter, and scoreboard update simultaneously. Vite's dev server starts in under 300ms and produces optimized bundles via tree-shaking.
🟩 Node.js 24 + Express
Backend server
Node.js handles thousands of WebSocket connections in a single thread via its non-blocking event loop — unlike traditional servers that spin up one thread per connection. Express handles HTTP routing and serves the compiled frontend from the same process.
🔌 Socket.io 4
Real-time WebSockets
HTTP is request-response — a game needs the opposite: the server must push updates to all clients on every event. Socket.io provides WebSockets with automatic fallback to long-polling and built-in room management for broadcasting to all players in a session.
🗄️ SQLite (node:sqlite)
Embedded database
SQLite runs serverless — the entire database is a single file on disk, with no separate process or credentials. Node.js 22+ includes native support via node:sqlite with no extra libraries. For short classroom sessions with no need for horizontal scaling, it offers the optimal complexity-to-functionality ratio.
🔊 Web Audio API
Sound synthesis
All sound effects are synthesized in real time in the browser — no .mp3 or .wav files needed. This eliminates HTTP requests for audio assets, guarantees offline functionality, and keeps the bundle size minimal.
☁️ Render Cloud
Deployment & hosting
Render provides free-tier Node.js hosting with automatic deployments from GitHub. The entire app — frontend + backend + database — runs as a single Node.js service, making deployment straightforward with no DevOps overhead.
