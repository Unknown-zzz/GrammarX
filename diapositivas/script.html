The Problem · The Solution — GrammarX's educational approach
Slides 3 – 4
≈ 1 minute · ~125 words
Slide 3 The Problem — advance slide, then speak
English grammar is often taught through static, repetitive exercises that students complete mechanically. There is rarely any immediate feedback — by the time a student learns they made a mistake, the lesson has already moved on. And because students work individually, there is no competitive element that creates urgency or genuine motivation to improve.
Slide 4 The Solution — advance slide, then speak
GrammarX transforms this dynamic completely. Instead of worksheets, students join a live competition from any device — no sign-up, no installation, just a browser. Every answer triggers instant feedback, and speed is rewarded with bonus points, so accuracy alone is not enough — you also have to be fast. The teacher stays in full control: they choose the game mode, set the time per round, and advance each question from their dashboard.
💡 Tips for Carla Your part is the emotional and educational core of the presentation — make the problem feel real.
On slide 3, slow down slightly — you want the audience to feel the problem before you offer the solution.
On slide 4, pick up the energy — the solution is exciting.
Emphasize "fast" at the end — it's a key game mechanic and it lands well.
Say "repetitive" as: re · PET · i · tiv. Say "urgency" as: UR · jen · see.

Speaker 3 — The game modes and how to play

The 4 Mini-Games · How to Play
Slides 5 – 6
≈ 1 minute · ~125 words
Slide 5 The 4 Mini-Games — advance slide, then speak
GrammarX includes four game modes, each targeting a different grammar structure. Time Machine tests Past Simple — words of a sentence are scrambled and you must tap them in the correct order. Duel Mode covers Comparatives and Superlatives — you choose the correct form from four options. Evidence File focuses on Present Perfect — a sentence with one grammar error is shown and you have to correct it. And Crystal Ball covers Future Tense — you choose between will, going to, or won't based on the context.
Slide 6 How to Play — advance slide, then speak
Using GrammarX is very simple. The teacher creates a room and a unique code plus a QR code are generated automatically. Students join from any device — no registration needed. The host then picks the game mode and the time per round, from ten to sixty seconds, and presses start. Everyone plays ten rounds simultaneously, and the player who answers fastest and most accurately wins. The podium is shown at the end, and the class can play again instantly with a different game mode.

Say "Comparatives" as: com · PAR · a · tivs. Say "Superlatives" as: soo · PER · la · tivs.

Speaker 4 — The technical system
JC
Jonathan Campos
System Architecture · Real-Time Sync · Tech Stack · Key Stats
Slides 7 – 10
≈ 1 minute · ~125 words
Slide 7 System Overview — advance slide, then speak
GrammarX is a Single Page Application — the entire frontend is a React component tree compiled by Vite and served as static files. The backend is a single Node.js process that handles both HTTP routes via Express and real-time WebSocket connections via Socket.io.
Slide 8 Architecture & Real-Time Sync — advance slide, then speak
All game state lives exclusively on the server. When any event occurs — a player answers, a round advances, the game ends — the server calls a central function that broadcasts the updated state to every socket in the room instantly. This means clients are completely stateless: if a player disconnects and reconnects, they receive the current state and resume seamlessly. The database is SQLite, which runs embedded inside the same Node.js process — no separate database server is required.
Slide 9 Tech Stack — advance slide, then speak
We chose React because it re-renders only the components affected by state changes, which is critical when a timer, an answer counter, and a scoreboard all update simultaneously. Node.js handles thousands of WebSocket connections in a single thread through its non-blocking event loop. Sound effects are synthesized in the browser via the Web Audio API, so no audio files are needed. The app is deployed on Render Cloud.
Slide 10 Key Stats — advance slide, then speak (then advance to Slide 11)
In total: four game modes, over forty questions each, five difficulty levels, and infinite unique sessions — all in a single codebase, with zero sign-ups required. You can try it right now at grammarx dot onrender dot com. Thank you — do you have any questions?
